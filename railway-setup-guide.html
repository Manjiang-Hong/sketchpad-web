🎯 Railway 环境变量设置（最新界面）

1 打开项目设置
  • 在 Railway 控制台，点击你的项目名称（例如：sketchpad-web）
  • 你会看到项目的概览页面
  • 在顶部找到 "Settings" 标签并点击
📱 点击顺序：项目名 → Settings 标签
2 找到 Variables 部分

在 Settings 页面向下滚动，找到以下部分：

Environment Variables

或者：

Variables

这个部分通常在页面中部或底部

3 添加环境变量

点击 "Add Variable" 按钮，然后添加：

第一个变量：
Name: JWT_SECRET
Value: sketchpad-production-secret-2026
第二个变量：
Name: PORT
Value: 3001

每添加一个变量后，点击 "Add" 或按回车保存

4 添加持久化存储（可选）
✅ 好消息！
代码已更新，现在 不需要手动配置 Volume！
数据会自动保存在正确的位置。

如果 Railway 提供了 Volume 选项，可以这样配置（可选）：

  • 在 Settings 页面继续向下滚动
  • 找到 "Volumes" 部分（如果存在）
  • 点击 "Add Volume"
  • 挂载路径：/app/server/data
注意：如果找不到 Volumes 选项，不用担心！
应用仍然可以正常运行，只是重启后用户数据可能会重置。
对于测试阶段，这完全可以接受。
5 重新部署

添加完环境变量和 Volume 后：

  • 回到项目的 "Deployments" 标签
  • 点击右上角的 "Redeploy" 按钮
  • 或者等待自动触发重新部署
✅ 部署成功后：
复制 Railway 提供的域名，类似：
https://sketchpad-web-production-abc123.up.railway.app
? 找不到 Variables？试试这个

方法 A：使用快捷方式

  • 在项目页面，按 Ctrl+K（Mac: Cmd+K）
  • 输入 "variables"
  • 选择 "Add Variable" 快速命令

方法 B：在服务页面添加

  • 点击项目中的具体服务（通常叫 "web" 或 "service"）
  • 在服务详情页，点击顶部的 "Variables" 标签
  • 这里可以直接添加环境变量

方法 C：使用 CLI

railway variables set JWT_SECRET=sketchpad-production-secret-2026
railway variables set PORT=3001
🎉 完成！
设置完成后，Railway 会自动重新部署应用。
记得把后端域名保存下来，等下部署 Vercel 前端时要用！
