2 部署后端到 Railway

Railway 是一个现代化的云平台，提供免费额度（$5/月），足够测试使用。

  • 访问 Railway： 打开 Railway
  • 点击右上角 "Login"，选择 "Login with GitHub"
  • 授权 Railway 访问你的 GitHub 账号
  • 登录后，点击 "New Project"
  • 选择 "Deploy from GitHub repo"
  • 找到并选择 "sketchpad-web" 仓库
⚠️ 重要：添加环境变量

在部署前，需要添加以下环境变量：

JWT_SECRET=sketchpad-production-secret-2026 PORT=3001

操作步骤：Settings → Variables → Add Variable

⚠️ 重要：配置持久化存储

Railway 默认每次部署会重置文件系统，必须添加 Volume 才能保存用户数据。

  • 在项目中点击 "Volumes" 标签
  • 点击 "Add Volume"
  • 挂载路径：/app/server/data
  • 点击 "Add Volume" 保存
✅ 部署成功后：

复制后端 URL（类似：https://sketchpad-web-production-abc123.up.railway.app）

下一步部署前端时需要用到这个地址！

3 部署前端到 Vercel

Vercel 是 React/Vite 项目的最佳部署平台，免费且自动提供 HTTPS。

  • 访问 Vercel： 打开 Vercel
  • 点击右上角 "Sign Up"，选择 "Continue with GitHub"
  • 授权 Vercel 访问你的 GitHub 账号
  • 登录后，点击 "Add New Project"
  • 找到 "sketchpad-web" 仓库，点击 "Import"
  • Framework Preset 会自动检测为 "Vite"
⚠️ 重要：配置环境变量

在部署前，需要添加后端 API 地址：

VITE_API_HOST=sketchpad-web-production-abc123.up.railway.app

操作步骤：Environment Variables → Add → Name: VITE_API_HOST → Value: 上一步复制的后端域名

注意：只填域名，不要带 https://

  • 点击 "Deploy"，等待构建完成（约 1-2 分钟）
  • 部署成功后，点击 "Continue to Dashboard"
  • 在 Dashboard 顶部可以看到访问链接（类似：https://sketchpad-web.vercel.app）
✅ 部署完成！

点击链接访问你的 SketchPad Web 应用 🎉

? 常见问题

Q: 用户数据丢失怎么办？

A: 检查 Railway Volume 是否正确挂载到 /app/server/data

Q: 前端无法连接后端？

A: 检查 Vercel 环境变量 VITE_API_HOST 是否正确设置（只填域名，不带 https://）

Q: 登录失败？

A: 检查后端日志（Railway 控制台），确认 JWT_SECRET 环境变量已设置

Q: 跨域错误？

A: 后端已配置 CORS，允许所有 .vercel.app 域名。如有问题，检查后端日志。
